## ADR Format

Each ADR follows this structure:

1. **Title** — Short descriptive name
2. **Status** — Proposed / Accepted / Deprecated / Superseded
3. **Date** — When the decision was made
4. **Context** — The issue motivating this decision
5. **Decision** — The change being proposed or made
6. **Consequences** — What becomes easier or harder as a result
7. **Implementation** — (Optional) Technical details and guidance
